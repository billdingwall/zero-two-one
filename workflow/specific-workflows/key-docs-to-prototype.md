# Flow: Key Docs to Prototype

**Scope:** Phase 1 & 2 (Planning & Pre-build)

## The Flow

1. **Ideation & Research:** Raw inputs enter `requirements/_notes/`.
2. **Drafting Key Docs:** The inputs inform the drafting of the living documents (`01-PRD.md`, `02-EDD.md`, `03-TDD.md`).
3. **Design System:** The `02-EDD.md` drives the creation of `DESIGN.md` (or a Design System) and visual assets (`requirements/_design/`).
4. **Prototype Generation:** The PRD logic and DESIGN tokens are used to build the static HTML/CSS/JS prototype in `prototype/`.
5. **Validation:** The prototype serves as the visual contract for stakeholders to react to. Feedback from this prototype feeds back into the Refinement Loop to update the Key Docs.
