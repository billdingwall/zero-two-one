# Flow: Review > Backlog > SSD

**Scope:** Continuous Improvement (Phase 4 Growth)

## The Flow

This workflow illustrates how user feedback and production analytics cycle back into the development pipeline.

1. **Review (`templates/05-REVIEW-Template.md`)**
   User analytics, feedback, and stakeholder requests are captured into a new refinement review round.
2. **Backlog (`04-BACKLOG.md`)**
   The review round synthesizes actionable enhancements, new features, and bug fixes directly into the backlog (while simultaneously updating the Living Docs if core logic/architecture changed).
3. **Spec-Driven Delivery (SSD)**
   The newly prioritized backlog items are pulled into the SSD pipeline (`specs/NNN-feature/`) for immediate, gated implementation.
