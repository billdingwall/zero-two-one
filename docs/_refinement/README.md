# Refinement Loop

This directory tracks the project-level documentation refinement loop.

## The Loop
1. **Review**: Add `r{n}-review.md` containing raw UX/functionality notes or direction notes (where `{n}` is the global round number).
2. **Synthesize**: Create `r{n}-update-{doc}.md` summarizing the section-by-section change list for the affected documents.
3. **Update**: Apply those changes to the actual documents (`docs/product-requirements.md`, `docs/technical-design.md`, etc.) and add changelog entries.
4. **Repeat**: Reflect updates in `DESIGN.md` and prototypes, then begin the next round.
