# Requirements Overview

The core documentation that defines the product.

## Manifest

- `01-PRD.md`: Product Requirements Document (What & why — business logic, feature requirements, data model).
- `02-TDD.md`: Technical Design Document (How - Technical — Architecture overview, data flow, locked decisions).
- `EDD.md`: Experience Design Document (How - Experience — UI/UX source of truth, interaction architecture, state definitions). Note: Specific design tokens are maintained in `DESIGN.md` in the repository root.
- `03-ROADMAP.md`: Phased plan and milestone gates.
- `04-BACKLOG.md`: Planned backlog and project tracker.
- `_design/`: Holds design assets.
- `_notes/`: Unstructured research, analysis and background context.
- `_refinement/`: Tracks the refinement loop cycles.
