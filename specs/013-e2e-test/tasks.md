## Phase 1 — Setup & guardrails
- [x] T001 **Pin the seams before writing code (no task changes these).** (a) **Artifact = the `package/` tarball** — `npm run sync:package -- --check` then `cd package && npm pack` → the 108-file shipped artifact; root/source `npm pack` is wrong (research R1). (b) **Toolbox topology** — install the tarball **once** into a throwaway toolbox; run installed bins against **pristine** targets via the **target-dir positional** (`init <target> --stack`), so targets carry no `node_modules` to trip migrate detection (research R2). (c) **Gate probe = `src/feature.js`** — the pre-commit hook ignores `*.md` and `^(specs|.kiro|requirements|workflow|skills|templates|prototype|hooks|scripts|bin|.github|.ai)/`, so a probe inside those is silently ignored and the test would falsely pass (research R3). (d) **Hook is live only after `git init`** — target is `git init`'d **before** install. (e) **`--phase planning`** so `021 qa` runs the docs tier green without app code (research R5). (f) **Engine-aware gate** — `github-speckit` (`specs/<feat>/spec.md`) for claude/antigravity, `kiro-specs` (`.kiro/specs/<feat>/requirements.md`) for kiro (research R4). (g) **Nothing ships** — `test/e2e/**` root-only, `test:e2e` in **root** `package.json` only, CI job under `.github/workflows/` (dropped by sync `githubExclusions`) (research R7).

## Phase 2 — Harness helpers (each encodes an acceptance assertion)
- [x] T003 **`packAndInstall()`** (FR-001/008; contracts): `sh('npm run sync:package -- --check', REPO_ROOT)` (fail fast on drift); `sh('npm pack', REPO_ROOT/package)` → resolve the emitted `zero-two-one-<ver>.tgz`; `toolbox = mkdtemp()` + minimal `package.json`; `sh('npm install --prefer-offline --no-audit --no-fund <tgz>', toolbox)`; return `{ toolbox, tgz, BIN_INIT, BIN_021, SPECKIT: require(join(toolbox,'node_modules/zero-two-one/speckit')) }`. **The heaviest step; gates the whole suite.**
- [x] T004 [P] **`provisionTarget(mode)`** (FR-002/006; research R8): `target = mkdtemp()`; `git init -q`; **local** `git config user.email/user.name`. `migrate` → seed **`docs/overview.md`** (a non-framework, **non-code** file — the migrate trigger that keeps phase planning) + a custom `CODE.md` (create-if-missing preservation) + an executable `.git/hooks/pre-commit` carrying a marked user body (e.g. `echo USER-HOOK`; chaining signal). `scaffold` → no extra seeds. **Do NOT seed `README.md`/`specs/_INDEX.md`/a `src/*.js`** — README/CODE.md are framework surface (won't trigger migrate), `specs/_INDEX.md` isn't a populated-specs signal, and a code file flips the detected phase to `mvp` so `qa` runs the code tier and fails *(analyze A2)*.
- [x] T005 [P] **`runInit` + `run021`** (FR-001/003): `runInit(BIN_INIT,target,stack,mode)` = `spawnSync('node',[BIN_INIT,target,'--stack',stack, ...(mode==='migrate'?['--yes']:[])])`, assert exit 0 — `--yes` on migrate is belt-and-suspenders; `spawnSync` has no TTY so the interview is already non-interactive *(analyze A5)*. `run021(BIN_021,target,args)` = `spawnSync('node',[BIN_021,...args],{cwd:target})` → `{code,stdout,stderr}` (project-scoped by `cwd`).
- [x] T006 [P] **`assertSurface(target,stack)`** (FR-002; Scenario 1/2/5): the per-stack signature files exist (data-model): claude → `CLAUDE.md` + `.claude/commands/021-status.md`; antigravity → `AGENTS.md` + `.agents/skills/021-status/SKILL.md`; kiro → `.kiro/steering/021-product.md` + `.kiro/agents/021.json`. Plus the shared Layer-1 core — assert the **full key-doc trio** `requirements/{01-PRD,02-EDD,03-TDD}.md` (so a `qa`-green cell can't mask a missing EDD/TDD *(analyze A6)*) + `hooks/pre-commit` + `.zero-two-one.json`.
- [x] T007 [P] **`assertManifest(SPECKIT,target,{stack,mode,ssd})`** (FR-003; Scenario 1/2): a **single** `SPECKIT.manifestFacts(target)` call → `{ stack, phase(='planning'), ssd, mode }` all match the cell — `manifestFacts` already carries `mode` (no separate `readManifest`) *(analyze A4)*. Read through the **shipped** parser — no output-scraping (also exercises the spec-009 `exports` map).
- [x] T008 [P] **`assertLifecycleGreen` + `assertNonDestructive`** (FR-003; Scenario 4/2): the former runs `run021('status')`/`('qa')`/`('doctor')` and asserts each exits 0 — `doctor` exits 0 on a fresh install because its non-zero path fires only on a **hard-severity** finding (advisory ⇒ 0; a clean planning scaffold has none) *(analyze A3)*; the latter (migrate only) asserts the seeded `docs/overview.md` **and** custom `CODE.md` are byte-identical and the original hook body (`USER-HOOK`) is still present in `.git/hooks/pre-commit` (chained, not clobbered).
- [x] T009 **`proveGate(BIN_021,target,stack)`** (FR-004; Scenario 3) — the fidelity core: **first make an initial commit on the default branch** (`git add -A && git commit -m init`) so **HEAD is born** — otherwise `git rev-parse --abbrev-ref HEAD` fails on the unborn branch, `currentBranch()`→null, the branch never resolves to a spec, and the gate can't evaluate *(analyze A1, verified)*; the hook skips the gate off a non-`NNN-` branch, so this initial commit is unblocked. Then `git checkout -b 042-e2e-probe`; **`writeSpec(target,stack,feat,'Draft')`** (engine-appropriate: github-speckit `specs/<feat>/spec.md` vs kiro-specs `.kiro/specs/<feat>/requirements.md`); write `target/src/feature.js` (**outside** the exclude surface, T001c); `git add -A`; **blocked:** real `git commit` → assert `code !== 0` && `/COMMIT BLOCKED/.test(stderr)`; `run021('spec status set 042-e2e-probe Approved')` (engine `writeStatus`); `git add -A` (stage the status change); **allowed:** real `git commit` → assert exit 0. **Verified** both engines: Draft blocks (exit 1), Approved allows (exit 0).
- [x] T010 [P] **`teardown(paths)`** (FR-006): `rmrf` each of toolbox, `tgz`, and every target — invoked from the suite's `after()` so it runs on pass **or** fail.
