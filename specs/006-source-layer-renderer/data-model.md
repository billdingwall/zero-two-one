# Data Model: Source Layer & Stack-Parameterized Renderer

*The states and artifacts the seam works with. Behavior in [plan.md](plan.md); the contract in [contracts/](contracts/). No manifest schema change — `tools.stack` already exists (TDD §7).*

## 1. StackAdapter registry (`scripts/init/adapters.js`)

One entry per stack; the single source `classes.js` and `sources.js` both consult (FR-006).

| Field | Type | Meaning |
|---|---|---|
| `entrypoint.template` | string (relpath under `templates/`) | Neutral source for the entrypoint doc — always `ASSISTANT-Template.md` |
| `entrypoint.dest` | string (target relpath) | Where the rendered entrypoint is written |
| `surfaceDirs` | string[] | Layer-2 command/skill dirs owned by this stack (framework-owned when chosen) |

| Stack | `entrypoint.dest` | `surfaceDirs` | Notes |
|---|---|---|---|
| `claude` (default) | `CLAUDE.md` | `['.claude/commands']` | Byte-identical to today (FR-010) |
| `antigravity` | `AGENTS.md` | `[]` | Entrypoint only; `.agents/skills/` tree = spec 007 |
| `kiro` | — | — | **Reserved** — populated by spec 008; `getAdapter('kiro')` **throws** (analyze A5), not a silent claude fallback |

`getAdapter(stack)` returns the entry, defaulting to `claude` when the stack is **absent/unknown** (FR-007); a **known-but-unpopulated** stack (`kiro`) throws (analyze A5).

## 2. The three layers as path sets

The install surface for a given `stack` is derived from these sets (repo-refactor §3.1):

| Layer | Paths | Class | Stack-varies? |
|---|---|---|---|
| **1 — neutral core** | `LAYER1_DIRS` = `scripts`, `hooks`, `skills`, `workflow`, `templates`, `.github`; `requirements/*` docs; `.ai/context`; manifest | framework / user / generated (per spec 001) | **No** — byte-identical for every stack, **except** `.zero-two-one.json` (records `tools.stack` + per-stack `files` inventory) and empty-scaffold `.ai/context` — excluded from the byte-identical assertion (analyze A1) |
| **2 — adapter** | `getAdapter(stack).surfaceDirs` + `getAdapter(stack).entrypoint.dest` | framework (surface dirs) / user (entrypoint doc) | **Yes** — only the chosen stack's paths |
| **3 — working-repo only** | `bin`, `specs`, `test`, `requirements/` (framework's own), CI, sync scripts | excluded from install surface | n/a — never ships/installs |

**Invariant (Scenario 4):** across two stacks, only Layer-2 paths may differ — excluding the manifest and empty `.ai/context` (analyze A1) and the merged files `package.json`/`.gitignore`, which carry target identity like the project name (analyze A7).

## 3. `classify(relPath, stack)` — stack-aware ownership

`classify` gains a `stack` parameter; behavior changes only for Layer-2 paths:

| Path | Under chosen stack | Under a different stack |
|---|---|---|
| `getAdapter(stack).surfaceDirs/**` (e.g. `.claude/commands/**`) | `framework-owned` | `null` (outside managed surface — never created/refreshed) |
| `getAdapter(stack).entrypoint.dest` (e.g. `AGENTS.md`) | `user-owned` (create-if-missing) | not in this install's mapping |
| Layer-1 paths | unchanged for every stack | unchanged for every stack |

`FRAMEWORK_DIRS(stack) = [...LAYER1_DIRS, ...getAdapter(stack).surfaceDirs]`.

## 4. Plan action: `render` vs `instantiate`

`userDocMappings(sourceDir, stack)` returns the doc-install mappings; the **entrypoint** mapping is tagged for rendering:

| Action | Applies to | Effect |
|---|---|---|
| `render` (NEW) | the stack's `entrypoint` (`ASSISTANT-Template.md` → `dest`) | `render.js` transforms the neutral source → entrypoint doc; preserves a user-owned local section on re-render (FR-003) |
| `instantiate` (spec 001, unchanged) | `CODE`, `PRODUCT`, `DESIGN`, `README`, `requirements/*` | verbatim copy of `templates/*-Template.md` (create-if-missing) |

Non-entrypoint guiding docs and `requirements/*` are **stack-invariant** (Layer 1).
